> **TL;DR**：服务器 = 大楼，端口 = 房间号，防火墙 = 门卫。门卫不让你进，你在外面喊破喉咙也没用。去云控制台的"安全组/防火墙"里把需要的端口加上，问题秒解决。

---

## 为什么明明装好了，网页却打不开？

新手最常见的困惑：

```
宝塔面板明明提示"安装成功，请访问 http://IP:8888" →
复制到浏览器 → 转了半天 → 无法访问
```

这不是你装错了。是云服务器出厂时，为了安全，**默认只开了 22 端口（SSH）**。其他所有端口——80、443、8888、5700——全部关着。

你需要手动去云厂商的控制台"放行"这些端口。

---

## 一张图理解端口

```
🏢 服务器 IP = 大楼地址（比如 47.118.28.27）
   ├── 🚪 22 号房  → SSH 远程连接（默认开着）
   ├── 🚪 80 号房  → 网站 HTTP（默认关着）
   ├── 🚪 443 号房 → 网站 HTTPS（默认关着）
   ├── 🚪 8888 号房 → 宝塔面板（默认关着）
   ├── 🚪 5700 号房 → 青龙面板（默认关着）
   └── 🚪 3306 号房 → MySQL 数据库（默认关着，强烈建议继续关着）
```

防火墙就是大楼门卫。你得告诉门卫"8888 号房可以进人"，访客才能进去。

---

## 操作步骤（阿里云为例）

### 轻量应用服务器

1. 进入 [轻量应用服务器控制台](https://swas.console.aliyun.com)
2. 点击你的服务器 → 左侧菜单「安全」→「防火墙」
3. 点「添加规则」，填：

| 字段 | 填什么 |
|------|--------|
| 协议 | TCP |
| 端口 | 8888（宝塔）/ 5700（青龙）/ 80（网站） |
| 来源 | 0.0.0.0/0（允许所有人访问） |

4. 保存。**立即生效，不用重启。**

### ECS 云服务器

1. 进入 [ECS 控制台](https://ecs.console.aliyun.com)
2. 点击实例 → 「安全组」→「配置规则」→「入方向」
3. 点「手动添加」，字段同上
4. 保存

> 💡 **其他云厂商**：腾讯云叫"安全组"，AWS 叫"Security Group"，Azure 叫"网络安全组"。名字不同，操作逻辑完全一样。

## 常用端口速查表

> ⚠️ 标记为 🔒 的端口强烈不建议对外开放，容易被扫描攻击。

| 服务 | 端口 | 说明 |
|------|------|------|
| SSH | 22 | 远程连接，**别关** |
| HTTP | 80 | 网站默认端口 |
| HTTPS | 443 | SSL 加密网站 |
| 宝塔面板 | 8888 | 建议安装后改成冷门端口 |
| 青龙面板 | 5700 | 薅羊毛神器 |
| **Docker 应用** | | |
| qBittorrent | 8080 | BT 下载器 WebUI |
| Jellyfin | 8096 | 媒体服务器 |
| Alist | 5244 | 网盘挂载 |
| Portainer | 9443 | Docker 可视化管理 |
| Uptime Kuma | 3001 | 网站监控 |
| **数据库（🔒 别对外开放！）** | | |
| MySQL | 3306 | 🔒 只允许 localhost |
| Redis | 6379 | 🔒 务必设密码 |
| PostgreSQL | 5432 | 🔒 同上 |
| MongoDB | 27017 | 🔒 默认无密码，最危险 |

## 放行了还打不开？四步排查法

### 第一步：确认端口在监听

```bash
netstat -ntlp | grep 8888
```

没输出 = 服务没启动或没绑这个端口。

### 第二步：检查系统级防火墙

云控制台放行了还不够，服务器系统内部也可能有防火墙：

```bash
# CentOS
systemctl status firewalld
# 临时关闭测试：
systemctl stop firewalld

# Ubuntu/Debian
ufw status
# 临时关闭测试：
ufw disable
```

> ⚠️ 这只是**排查手段**，排查完记得开回来。

### 第三步：确认服务绑定地址

有些服务默认只绑 `127.0.0.1`（仅本机访问），需要改为 `0.0.0.0`（允许外部访问）。检查服务的配置文件。

### 第四步：用 telnet 测试

```bash
# 在你自己的电脑上运行（不是服务器）
telnet 47.118.28.27 8888
```

- 显示 `Connected` = 端口通了 ✅
- 一直 `Trying...` = 端口不通 ❌

---

## 安全建议

- 宝塔面板的 8888 端口建议改成别的冷门端口（5 位数的，比如 28888）
- MySQL 的 3306 **绝对不要**对外开放，被扫到就是暴力破解
- Redis 的 6379 如果不设密码就开放，基本上等于把服务器钥匙挂门口

---

## 总结

端口放行是玩服务器的第一课。记住这个排查流程：

1. 访问不了 → 先检查云控制台防火墙
2. 防火墙 OK → 检查系统内部防火墙
3. 都 OK → 检查服务是否在监听
4. 都在监听 → 用 telnet 测试连通性

99% 的问题会在第一步解决。
