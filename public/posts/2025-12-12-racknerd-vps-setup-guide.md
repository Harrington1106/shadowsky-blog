> **简介**：RackNerd 以其极低的价格（年付 $10-$20 左右）和相对稳定的美国西海岸线路，成为了学生党和个人开发者的首选。但由于其反欺诈系统较为严格，新手很容易在注册环节遭遇“砍单”。

## 第一阶段：注册与购买（通过风控的关键）

很多用户在下单后会立即收到 "Fraud Check"（欺诈检测）的通知，导致订单取消。要避免这种情况，必须遵循**“真实性原则”**。

### 1. 网络环境准备
*   **严禁使用代理**：注册时**必须关闭**任何 VPN 或代理工具。系统检测到你的 IP 地址（如美国）与你的支付方式（支付宝/中国）或注册资料（中国地址）不符时，会直接判定为盗刷信用卡的欺诈行为。
*   **使用国内 IP**：直接使用本地网络访问 RackNerd 官网。

### 2. 资料填写规范
在填写注册信息（Client Information）时，请注意以下细节：
*   **Country (国家)**：必须选择 **China**。
*   **Address (地址)**：可以使用拼音填写你所在的城市（如 `Beijing`, `Shanghai`），无需精确到门牌号，但不要瞎填美国地址。
*   **Phone Number**：填写真实的 `+86` 手机号。
*   **Email**：推荐使用 Gmail 或 Outlook。QQ 邮箱有时会拦截官方的开通邮件。

### 3. 支付方式
RackNerd 对国人非常友好，直接选择 **Alipay (支付宝)** 即可。这不仅方便，也是证明你真实身份的一个辅助要素。

## 第二阶段：获取服务器信息

支付成功后，不要在那个复古的 SolusVM 控制面板里浪费时间寻找密码。RackNerd 的面板功能较弱，主要用于重装系统或重启。

**正确获取密码的方式是查看邮件：**

1.  登录你的注册邮箱。
2.  查找标题包含 **"KVM VPS Login Information"** 的邮件。
3.  邮件中包含了你最重要的资产信息：
    *   **Main IP**: 服务器公网 IP。
    *   **Root Password**: 初始 Root 密码（通常是一串随机字符）。
    *   **Control Panel**: 控制面板地址（平时很少用到，除非需要重装系统）。

## 第三阶段：SSH 连接与常见错误排查

获得 IP 和密码后，推荐使用 **FinalShell**、**Xshell** 或 **Termius** 等专业工具进行连接。

### 1. 密码输入无反应（Linux 特性）
如果你使用 Windows 自带的 PowerShell 或 CMD 连接：
*   当你输入密码时，**屏幕上不会显示任何星号或圆点**，光标也不会移动。
*   这是 Linux 的安全机制。请放心输入（或右键粘贴），然后直接按回车。

### 2. "Permission denied" 错误
这通常是密码复制错误导致的。
*   **检查空格**：邮件中的密码前后容易复制到多余的空格。建议先复制到记事本中确认无误，再复制到 SSH 客户端。
*   **区分大小写**：Linux 密码严格区分大小写。

### 3. 连接成功后的第一步
成功登录（出现 `root@... #` 提示符）后，你已经拥有了服务器的最高权限。

此时，你可以根据自己的需求进行后续操作：
*   **搭建代理环境**：请参考 **[阿里云轻量服务器搭建美国原生IP环境全流程指南](/posts/2025-12-08-alibaba-cloud-us-ip)** 中的“一键部署服务”部分（脚本是通用的，适用于 Debian/Ubuntu 系统）。
*   **部署网站或 Docker**：请参考 **[服务器玩法大全](/posts/2025-12-08-server-usage-guide)**。

## 必读：关于速度与晚高峰优化

由于 RackNerd 走的是普通线路（ChinaNet 163），在晚高峰（20:00 - 23:00）不可避免地会出现拥堵。

### 1. 开启 BBR (救命稻草)
对于 RackNerd 这种非 CN2 线路，BBR 是必选项。
登录 SSH 后执行：
```bash
echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p
```
验证：`lsmod | grep bbr`，看到 `tcp_bbr` 即成功。

### 2. 协议选择建议
*   **Shadowsocks/VMess**：白天可用，晚高峰可能会严重丢包。
*   **Hysteria 2 (强烈推荐)**：基于 UDP 协议，在 RackNerd 这种丢包严重的线路上有奇效，能暴力抢占带宽，极大提升 YouTube 4K 播放流畅度。建议作为晚高峰的备用方案。
    *   👉 **[点击查看：Hysteria 2 一键部署实战教程](/posts/2025-12-12-hysteria2-guide)** (附全自动脚本，3分钟搞定)

### 3. 双协议共存策略 (最佳实践)
**Q: 两个协议可以随时切换吗？**
**A: 完全可以，且强烈建议同时部署。**

*   **原理**：你可以在同一台服务器上同时运行 Shadowsocks (例如占用 50000 端口) 和 Hysteria 2 (例如占用 40000 端口)。它们互不冲突，就像你的电脑可以同时打开 QQ 和微信一样。
*   **配置**：在客户端（如 v2rayN）中添加这两个节点。
*   **使用场景**：
    *   **白天 (9:00 - 19:00)**：优先使用 **Shadowsocks**。它的协议更轻量，手机端耗电更少，连接建立也更稳。
    *   **晚高峰 (20:00 - 23:00)**：如果发现 Shadowsocks 开始卡顿或断流，鼠标一键切换到 **Hysteria 2** 节点，瞬间恢复速度。

## 进阶技巧：如何重装系统

如果你把服务器“玩坏了”，或者想更换系统版本（如从 CentOS 换到 Debian）：

1.  登录 RackNerd 发送的邮件中的 **Control Panel** 链接。
2.  输入邮件中单独提供的面板账号密码（注意：这通常与服务器 Root 密码不同）。
3.  在面板中找到 **"Reinstall"** 按钮。
4.  选择你想要的系统（推荐 **Debian 11** 或 **Ubuntu 20.04**），点击重装。
5.  **注意**：重装后会生成一个新的 Root 密码，记得保存。
