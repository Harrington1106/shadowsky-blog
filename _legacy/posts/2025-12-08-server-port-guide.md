# 宝塔/青龙面板打不开？手把手教你服务器放行端口（防火墙设置）

很多新手在服务器上安装完宝塔面板、青龙面板或者 Docker 容器后，明明显示“安装成功”，但用浏览器访问 `http://IP:8888` 却怎么也打不开。

这通常不是你没装好，而是被服务器的 **“防火墙” (安全组)** 给挡住了。
云服务器为了安全，默认只开放 22 (SSH) 等极少数端口。想访问别的服务，必须手动“放行”。

## 核心概念：什么是端口？

如果把服务器比作一栋**大楼**，IP 地址就是**大楼的地址**。
而 **端口 (Port)** 就是大楼里的**房间号**。

*   **宝塔面板**：通常住在 **8888** 号房。
*   **青龙面板**：通常住在 **5700** 号房。
*   **Web 网站**：通常住在 **80** (HTTP) 和 **443** (HTTPS) 号房。
*   **SSH 远程连接**：住在 **22** 号房。

如果大楼的门卫（防火墙）不放行，你就在外面喊破喉咙也进不去。

## 操作步骤：在云厂商控制台放行 (以阿里云为例)

这是最关键的一步，99% 的无法访问都是因为这里没配置。

1.  **登录控制台**：进入阿里云/腾讯云官网，登录后找到 **“ECS 云服务器”** 或 **“轻量应用服务器”** 控制台。
2.  **找到安全组/防火墙**：
    *   **轻量应用服务器**：点击服务器卡片 -> 左侧菜单 **“安全”** -> **“防火墙”**。
    *   **ECS 云服务器**：点击实例 -> **“安全组”** -> **“配置规则”** -> **“入方向”**。
3.  **添加规则**：
    点击 **“添加规则”** 或 **“手动添加”**，填写以下信息：
    *   **协议类型**：选择 **TCP** (大部分服务都是 TCP)。
    *   **端口范围**：填你要开放的端口。
        *   宝塔填：`8888`
        *   青龙填：`5700`
        *   全部开放填：`1-65535` (方便但不太安全，测试时可用)
    *   **授权对象/来源 IP**：填 `0.0.0.0/0` (表示允许全世界任何人访问)。
4.  **保存**：点击确定/保存。

无需重启服务器，规则通常立即生效。再试着刷新浏览器，应该就能打开了！

## 进阶：常用端口对照表

在折腾服务器时，建议把这些常用端口记下来（或者收藏本文）：

| 服务名称 | 默认端口 | 说明 |
| :--- | :--- | :--- |
| **SSH** | 22 | 远程连接服务器，**千万别关** |
| **HTTP** | 80 | 网站默认端口 |
| **HTTPS** | 443 | 网站加密端口 |
| **宝塔面板** | 8888 | 安装完成后建议修改为其他冷门端口 |
| **青龙面板** | 5700 | 薅羊毛神器 |
| **MySQL** | 3306 | 数据库 (不建议对外开放，容易被黑) |
| **Redis** | 6379 | 缓存服务 (务必设置密码) |
| **Alist** | 5244 | 网盘挂载神器 |
| **Jellyfin** | 8096 | 媒体服务器 |
| **qBittorrent**| 8080 | 下载器 Web 界面 |

## 常见问题 (FAQ)

**Q: 我在云控制台放行了，为什么还是打不开？**
**A:** 
1.  **检查宝塔面板自身**：在 SSH 终端输入 `bt`，选择 `14` 查看面板默认信息，确认端口没变。
2.  **服务器内部防火墙**：有些系统镜像默认开启了 `ufw` 或 `firewalld`。
    *   尝试运行命令关闭内部防火墙（CentOS）：`systemctl stop firewalld`
    *   Ubuntu/Debian：`ufw disable`
3.  **服务没启动**：输入 `netstat -ntlp` 查看端口是否在监听列表中。

**Q: 直接开放所有端口 (1-65535) 有危险吗？**
**A:** 有一定风险。这意味着你服务器上运行的所有网络服务都暴露在公网。建议只开放需要的端口。如果必须全开，请确保你的 MySQL、Redis 等服务都设置了**强密码**。

## 总结

服务器“打不开”，第一反应先看**端口放行没**。
掌握了配置安全组规则，你就掌握了通往服务器各种精彩应用的大门钥匙。
