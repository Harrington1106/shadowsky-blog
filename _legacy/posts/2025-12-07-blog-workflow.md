## 👋 嗨，大家好！

经常有朋友问我：“Thoi，你的博客看起来好酷，平时写文章麻不麻烦呀？” 

哈哈，其实一点也不麻烦！甚至可以说，这是一种享受。今天我就把我的“独家秘籍”——也就是我的写作发布工作流（Workflow），毫无保留地分享给大家。搬好小板凳，我们要开始啦！✨

### 🌟 第一步：灵感乍现

一切的开始，通常都是在洗澡、散步或者发呆的时候（在这个充满 bug 的世界里，发呆也是一种生产力嘛）。

当我捕捉到一个有趣的想法，或者是解决了一个挠破头的技术难题，我会立刻打开我的备忘录记下来。比如：“写一篇关于博客发布流程的文章，语气要轻松一点！”

### 📝 第二步：创建 Markdown 文件

好啦，灵感有了，接下来就是动手。我的博客是基于 **Markdown** 驱动的，这意味着我不需要在这个复杂的后台点来点去，只需要创建一个简单的文本文件。

通常我会打开我的代码编辑器（VS Code YYDS！），在 `_legacy/posts` 目录下新建一个文件。文件名通常是这样的格式：

`YYYY-MM-DD-title.md`

比如这篇文章的文件名就是 `2025-12-07-blog-workflow.md`。看着这个文件名，是不是感觉特别有条理？😎

### 🎨 第三步：填充“元数据” (Front Matter)

打开文件，第一件事不是写正文，而是写“头文件”。这个叫 **Front Matter**，它是告诉博客系统这篇文章叫什么、是谁写的、发在哪个分类下。

就像这样：

```yaml
---
title: 嘿，我是怎么把这篇文章变出来的？
date: 2025-12-07
category: 博客运维
author: Thoi
tags: workflow, blog, fun
excerpt: 这是一段简短的摘要，用来吸引你点进来看的~
---
```

这两条虚线中间的内容，就是文章的“身份证”啦。

### ✍️ 第四步：沉浸式写作

接下来就是最开心的环节——写作！

因为是 Markdown 格式，我可以专注于文字本身，不用操心排版。
- 想加粗？用 `**` 包起来。
- 想贴代码？用 ` ``` ` 包起来。
- 想放图片？直接扔链接。

整个过程就像是在写代码，但比写代码轻松多了（毕竟不会报错嘛 😂）。

### 🚀 第五步：推送到云端

文章写好了，怎么让大家看到呢？

其实我的博客是托管在 GitHub 上的。所以，发布文章的过程，本质上就是一次代码提交。

打开终端，输入那几行熟悉的命令：

```bash
git add .
git commit -m "Add new post: Blog Workflow"
git push
```

随着进度条跑完，我的任务就结束了。

### ✨ 第六步：自动部署魔法

接下来的事情，就交给自动化的魔法了。GitHub Actions 会监测到我的更新，自动构建、部署。几分钟后，你刷新我的博客首页，这篇文章就会安安静静地躺在那里，等待着你的点击。

### 🎉 总结

看吧，这就是我的博客发布全流程：
1. **捕捉灵感** 💡
2. **新建文件** 📄
3. **填写信息** 🏷️
4. **快乐写作** ⌨️
5. **Git 推送** 🚀

简单、纯粹、高效。这就是我热爱这个博客系统的原因。
