# 首页首屏高级化收口设计文档

## 概述
- 项目：`shadowsky-blog`
- 日期：`2026-06-11`
- 主题：首页首屏 UI 突兀感优化
- 范围：`index.html`、`css/style.css`
- 页面目标：首页首屏从“梦幻科技 + 自我介绍 + 内容入口混搭”收口为“更克制、更高级的氛围封面”

## 设计前提
- 当前首页首屏已经接入共享公共壳层和液态玻璃样式系统。
- 现状问题不在于功能缺失，而在于视觉语言混杂：导航、背景、头像、主标题、标签、内容入口各自使用不同强度的表达方式，导致页面显得拼接感较强。
- 本次任务不扩散到全站，也不调整首页的功能逻辑，仅收口首页首屏气质。

## 用户确认结论
- 删除“突出我是谁 / 我有什么内容”的表达层。
- 删除首页首屏下方整块入口区，包括：
  - `Content Gateway`
  - “从这里进入我的内容宇宙”
  - 定位 / 入口 / 阅读方式摘要带
  - 四张入口卡片
- 首页首屏保留元素：
  - 导航
  - 头像
  - 主标题
  - 副句
  - 底栏
- 底栏保留，不删除。
- 气质方向选择：更高级、更克制，但保留现有站点的梦幻科技氛围。

## 当前问题归纳

### 1. 层级过多
- 当前首屏同时承载头像、自我标签、长副句、底栏和下方内容入口区，视觉任务过多。
- 首页既像个人介绍页，又像站点频道入口，又像一张氛围海报，角色不够清晰。

### 2. 视觉强度不统一
- 主标题体量和字重非常强，接近海报标题。
- 背景光斑和粒子是柔性的。
- 标签胶囊和副文案又非常轻。
- 多组元素的表达强度不在同一系统里，因此会显得突兀。

### 3. 结构断层明显
- 首屏结束后紧接一个功能性很强的内容入口区，打断了封面气质。
- 用户刚建立首页印象，就被导流卡片切换到另一种信息结构。

### 4. 高级感被“解释性内容”稀释
- 高级的首页封面通常依赖克制、留白和少量高质量信息。
- 当前首页包含较多说明性标签和入口信息，使首屏从“封面”变成“说明书”。

## 目标
- 让首页首屏成为单一、完整、统一的视觉封面。
- 保留现有梦幻科技感，但显著降低突兀感。
- 让头像、标题、副句、底栏形成单一阅读路径。
- 删除不必要的说明性内容和功能导流内容，让首屏更安静。

## 非目标
- 不改首页导航功能和移动菜单逻辑。
- 不改访问量统计逻辑。
- 不改背景粒子脚本行为，只调整其视觉承载方式。
- 不新增新的首页功能模块。
- 不重做整个网站品牌系统。

## 方案比较

### 方案 A：极简静态封面
- 做法：大幅减少背景表现、显著缩小头像和标题、副句极短，仅保留必要元素。
- 优点：最安静、最高级。
- 缺点：过度克制后可能显得偏空，且会损失当前网站已有的气质识别度。

### 方案 B：高级氛围封面（采用）
- 做法：保留梦幻科技背景，但降低噪声；头像、主标题、副句形成紧凑垂直构图；底栏保留为低调签名信息；删除所有解释性入口内容。
- 优点：兼顾高级感与站点原有风格，适合渐进式优化。
- 缺点：需要细调尺寸、透明度、留白和对比度，不能简单粗暴删减。

### 方案 C：品牌海报封面
- 做法：保留甚至强化主标题冲击，把首屏做成强视觉海报。
- 优点：记忆点强。
- 缺点：与用户当前要求的“更高级、更克制”不完全一致。

## 采用方案
- 采用方案 B：高级氛围封面。

## 页面结构设计

### 首屏结构
- 首屏只保留一个 `hero` 结构，不再拆成“首屏介绍”与“下方入口区”两段。
- 内容顺序固定为：
  1. 头像
  2. 主标题
  3. 副句
  4. 底栏
- 删除标签胶囊区。
- 删除 `home-gateway` 段落及其所有子内容。

### 视觉阅读路径
- 视线从导航向下，先看到头像，再落到主标题，再通过副句完成情绪收束，最后在底栏结束。
- 这样首屏只承担“建立印象”的职责，不承担“解释内容结构”的职责。

## 视觉设计

### 1. 导航
- 保留浮层导航位置和行为。
- 降低导航的厚重感：
  - 减少白底厚度感
  - 弱化边界和高光
  - 降低与首屏主内容的竞争
- 导航仍然是功能层，但不再成为首屏最醒目的横向元素。

### 2. 背景
- 保留粒子网络和柔和彩色光斑。
- 降低背景噪声：
  - 弱化黄色与粉紫大面积光斑的存在感
  - 保持大面积明亮、柔和、通透
  - 避免背景像一张“壁纸”压在内容后面
- 背景定位为“氛围层”，不是“主视觉层”。

### 3. 头像
- 保留头像作为个人识别锚点。
- 视觉上从“中心主视觉”调整为“标题前奏”：
  - 尺寸略缩小
  - 外发光和阴影强度下降
  - 减少悬浮感
- 在线状态点可保留，但应更干净精致。

### 4. 主标题
- 保留两行标题：
  - `探索未知`
  - `记录美好`
- 但做以下收口：
  - 略微降低字号
  - 略微降低字重压迫感
  - 收紧行距和上下间距
  - 保留渐变，但降低饱和度和荧光感
- 标题要像“高级封面标题”，而不是“宣传海报口号”。

### 5. 副句
- 副句保留，但从当前偏长、偏散的说明文本，收为一句更短、更稳、更安静的话。
- 副句职责：
  - 缓冲头像与标题的强度
  - 让标题有情绪落点
  - 与底栏形成精致的下行节奏
- 副句视觉应：
  - 宽度更窄
  - 行数更少
  - 颜色更稳
  - 与标题形成明确的主次关系

### 6. 底栏
- 保留底栏。
- 将底栏视为封面签名区，而非功能提示区：
  - 更薄
  - 更轻
  - 与正文保持足够距离
  - 不抢视觉焦点

## 删除清单
- 删除标签胶囊组。
- 删除首页 `home-gateway` 区域。
- 删除“定位 / 入口 / 阅读方式”摘要带。
- 删除“从这里进入我的内容宇宙”标题和说明。
- 删除四张入口卡片。

## 实施边界
- 只修改：
  - `index.html`
  - `css/style.css`
- 不调整：
  - 导航交互逻辑
  - 粒子脚本
  - 底栏访问量逻辑
  - 首页右键菜单逻辑

## 验收标准
- 首页首屏不再出现“自我介绍 + 内容导流”的混搭感。
- 首屏视觉层级清晰，阅读路径稳定。
- 主标题不再过于压迫，和背景、头像、副句关系更协调。
- 删除下方内容入口区后，首页仍然完整，不显空、不显断。
- 底栏保留且更自然，不喧宾夺主。
- 桌面端和移动端都保持单一封面气质。

## 风险与控制
- 风险：删除入口区后首页信息量下降，显得太空
  - 控制：通过更准确的留白、标题比例和副句节奏来撑住页面，而不是再补新模块
- 风险：背景弱化后首页失去识别度
  - 控制：保留粒子和柔色光斑，只降噪不清空
- 风险：标题削弱后缺乏记忆点
  - 控制：保留两行主标题和渐变，只收口强度，不推翻核心表达

## QA 审计报告

| 隐患点 | 严重程度 | 修复建议 |
|---|---|---|
| 删除入口区后首屏可能过空 | 中 | 用更精确的纵向留白和副句长度控制首页完整度 |
| 背景与主标题若降噪不平衡，可能从“突兀”变成“寡淡” | 中 | 先收标题强度，再同步降低背景光斑饱和度 |
| 本次不涉及 PHP 接口、SQL 写路径 | 低 | 无新增 SQL 注入风险 |
| 本次不涉及 Admin 鉴权逻辑 | 低 | 无新增 Token 校验风险 |
| 粒子背景与毛玻璃叠加仍可能影响低性能设备滚动体验 | 低 | 控制模糊和发光层数量，不增加新的高成本特效 |

QA 结论：方案为低风险，可进入实施计划阶段。

## 结论
本次首页优化不再强化“内容说明”或“频道导流”，而是把首页收口成一个真正的视觉封面。通过保留导航、头像、主标题、副句和底栏，删除标签与入口区，并降低背景和标题之间的冲突强度，首页将获得更统一、更克制、更高级的呈现。
