# 公开页导航液态玻璃与文案统一设计文档

## 概述
- 项目：`shadowsky-blog`
- 日期：`2026-06-11`
- 主题：公开页导航液态玻璃强化与导航文案统一
- 范围：公开前台页面导航、移动端菜单、相关共享样式
- 涉及文件：`css/style.css`、公开页 HTML、`js/core-pages-ui.test.js`、必要时 `docs/CODE_WIKI.md`

## 设计前提
- 当前公开页已经接入一套玻璃化导航与移动菜单样式，但整体更接近普通 glassmorphism，而不够像具有厚度、折光和环境取色感的“液态玻璃”。
- 当前导航文案在多个公开页中重复硬编码，桌面端 `title`、移动端菜单可见文本、部分 `aria-label` 口径并不统一。
- 仓库中未发现 `docs/DEVELOPMENT_MANUAL.md` 与 `project-state.yml`，本设计以当前代码和 `docs/CODE_WIKI.md` 为准。
- 用户已明确指出：此前命名建议没有贴合网站真实内容，尤其 `ACG` 入口实际上聚合了追番、漫画、剪辑和收藏视频，不能被误判为单纯“视频”栏目。

## 用户确认结论
- 视觉方向：采用更强的液态玻璃方向，但优先使用现有 CSS 能力强化导航材质感，不在本轮引入重型 WebGL 依赖。
- 文案范围：采用方案 A，只逐页同步替换公开页导航相关文案，不顺手抽离成共享配置。
- 顶层导航最终命名统一为：
  - 首页
  - 笔记
  - 片刻
  - 收藏
  - 订阅
  - ACG
  - 关于
- `ACG` 入口名称已经确认，原因是该入口真实承载番剧、漫画、剪辑与收藏视频的聚合职责。

## 当前问题归纳

### 1. 导航材质更像“普通玻璃”，不像“液态玻璃”
- 现有导航主要依赖半透明背景、模糊、描边和阴影。
- 这种表达可以形成玻璃感，但缺少液态玻璃应有的厚度、折光、高光漂移和环境染色。
- 导航 active 态与 hover 态目前更像配色变化，而不是材质变化。

### 2. 导航文案没有真正贴住网站内容
- 公开页导航文案长期沿用“星空笔记 / 随手拍 / 收藏 / 订阅 / 视频”等旧口径。
- 用户明确指出其中一部分概括并不准确，尤其 `ACG` 入口若继续被描述为“视频”会误导栏目定位。
- 站点是“个人记录 + 内容策展 + ACG 聚合”混合气质，命名既不能太功能化，也不能脱离真实页面职责。

### 3. 文案分散硬编码，逐页不一致风险高
- 导航文字、`title`、移动端菜单文本分散写在各个公开页 HTML 中。
- 只改局部页面会产生站内口径分裂。
- 本轮虽然不抽配置，但必须一次性定义统一替换标准。

## 目标
- 让公开页导航更接近液态玻璃的材质表现，而不是普通透明卡片。
- 让桌面导航、移动端菜单和交互状态使用统一的材质语言。
- 让公开页导航命名统一为贴合站点内容的稳定口径。
- 让 `title`、移动端文字和 `aria-label` 一致表达栏目真实职责。

## 非目标
- 不修改后台导航与后台文案。
- 不在本轮将导航组件重构成共享模板或单一配置源。
- 不改公开页主内容区标题、段落或频道正文结构。
- 不在本轮引入 WebGL/SVG 重型折射渲染方案。
- 不对全站所有卡片同时进行液态玻璃重做，仅聚焦导航与移动菜单。

## 方案比较

### 方案 A：轻玻璃微调
- 做法：保留现有导航结构，只提升透明度、模糊和描边。
- 优点：风险低，落地快。
- 缺点：会更精致，但仍然不像真正的液态玻璃。

### 方案 B：中度液态玻璃增强
- 做法：使用 CSS 增强导航表面高光、环境染色、active 岛屿感和 hover 的材质位移。
- 优点：在不引入复杂依赖的前提下，能显著靠近液态玻璃气质。
- 缺点：需要仔细控制日间/夜间分层，避免再次互相污染。

### 方案 C：强液态玻璃方向（采用）
- 做法：以方案 B 为基础，进一步强化导航胶囊的厚度、高光边缘、内层折光和 active 流体感，同时统一桌面与移动端的材质语言。
- 优点：最符合用户当前对“更像液态玻璃”的期待。
- 缺点：需要更严格地隔离明暗主题色温和材质规则。

## 采用方案
- 采用方案 C：强液态玻璃方向，但以 CSS 为主完成，不引入重型运行时依赖。

## 液态玻璃视觉设计

### 1. 材质目标
- 导航要像一整块具有厚度的半透明壳，而不是单纯模糊背景。
- 应具备三个层次：
  - 外轮廓与边缘描边
  - 顶部/侧边高光
  - 内层 active 岛屿或受压高亮区

### 2. 日间模式
- 以冷白蓝为主色温。
- 重点强化：
  - 顶部冷白高光
  - 边缘浅蓝描边
  - 轻微环境染色
  - hover 态的亮面位移感
- 避免再次退回普通白色半透明条。

### 3. 夜间模式
- 保持深色基底，但要保留液体材质的厚度感。
- 重点强化：
  - 暗部边缘高光
  - 内层深色折射
  - 独立阴影系统
- 日间冷白规则必须严格隔离，不能污染夜间。

### 4. Active 态
- active 态不只是改字色或背景色。
- 目标是做成“被液体包裹的高亮岛”：
  - 边缘更亮
  - 内部更通透
  - 与底板形成明显材质层级差

### 5. Hover 态
- hover 态应表现出轻微流体形变感，而不是普通按钮 hover。
- 允许使用：
  - 边缘亮度抬升
  - 内层高光位置变化
  - 阴影轻微变化
- 不使用过分夸张的位移动画，以免破坏克制感。

## 验收标准
- 日间导航看起来更像液态材质，而不是普通玻璃条。
- 夜间导航保持独立深色液态玻璃外观，不再受到日间色温污染。
- 桌面导航、移动端菜单和 active 态属于同一套材质语言。
- 全部公开页导航命名统一为：
  - 首页 / 笔记 / 片刻 / 收藏 / 订阅 / ACG / 关于
- `ACG` 入口的文案说明清楚对应追番、漫画与剪辑，不再被写成“视频”。
- 页面间 `title`、移动端菜单文字和 `aria-label` 不再互相冲突。

## 风险与控制
- 风险：强液态玻璃做过头，造成导航喧宾夺主
  - 控制：液态感体现在材质，而不是过强动画和过高饱和度
- 风险：日间/夜间选择器再次互相污染
  - 控制：将色温与阴影拆分为公共骨架、日间专属、夜间专属三层
- 风险：逐页改文案容易漏页面
  - 控制：通过测试和清单覆盖所有公开页入口文件
- 风险：本轮不抽配置，后续改字仍需多页同步
  - 控制：本次先完成统一，后续如有需要再单独做共享配置重构

## 结论
本次工作不是简单“改几个导航标题”，而是同时完成两件事：一是让公开页导航真正接近液态玻璃的材质表达，二是让导航命名回到网站真实内容之上。通过在 CSS 层面强化厚度、高光、折光和主题隔离，并在公开页逐页统一“首页 / 笔记 / 片刻 / 收藏 / 订阅 / ACG / 关于”这套命名，站点导航将同时获得更准确的栏目表达和更统一的品牌观感。
