# 核心公开页 UI 美化设计文档

## 概述
- 项目：`shadowsky-blog`
- 日期：`2026-06-11`
- 范围：`index.html`、`blog.html`、`post.html`、`moments.html`、`bookmarks.html`、`rss.html`
- 不包含：`admin/` 后台、接口逻辑改造、数据结构调整
- 设计方法：基于 `ui-ux-pro-max` 的 `Content First + Editorial + Controlled Glassmorphism`

## 背景
当前前台页面已具备一套液态玻璃基础能力，且仓库内已有 [全站统一液态玻璃视觉系统 Spec](file:///d:/Projects/shadowsky-blog/.trae/specs/unify-liquid-glass-ui/spec.md)。但核心页面之间仍存在以下问题：

- 页面首屏表达差异大，频道之间缺少统一骨架
- 液态玻璃元素已经存在，但使用强度不均匀，部分页面偏重装饰、部分页面偏工具感
- 导航、搜索区、工具条、卡片、页脚存在重复样式与局部风格分叉
- 内容密度较高的页面缺少更强的编辑感层级，影响阅读与浏览节奏

本次设计目标不是把所有页面做成同一个模板，而是在统一设计系统下，让各频道拥有各自清晰的内容气质。

## 目标
- 建立六个核心公开页的统一视觉骨架
- 在保留液态玻璃元素的前提下，提升内容优先级与可读性
- 将页面体验从“单页漂亮”提升为“全站属于同一内容品牌”
- 优先通过公共样式层和少量结构调整完成改造，避免大规模重写业务逻辑

## 非目标
- 不改 `admin/` 后台视觉
- 不重写数据加载、搜索、筛选、分页等业务逻辑
- 不引入新的大型前端依赖
- 不将正文阅读区做成重度玻璃化界面

## 设计原则

### 1. 内容优先
- 页面首先服务于阅读、浏览、筛选与发现
- 装饰效果不能抢占正文、标题、摘要和列表信息的层级

### 2. 编辑感骨架
- 页面结构参考 `Editorial / Magazine`
- 强调标题层级、留白、信息分组与内容预览
- 首页与频道首页是内容入口，内容页是阅读面板，工具页是工作台

### 3. 受控液态玻璃
- 玻璃态用于增强空间感与统一品牌感
- 强玻璃仅用于：导航、移动菜单、工具条、浮层、弹层、重点摘要卡
- 普通内容卡与正文主体只保留轻玻璃或轻边框，不使用重模糊

### 4. 一致而不单调
- 六页共用导航体系、容器系统、卡片规则与交互反馈
- 六页通过模块编排、色彩重点和内容组织体现频道差异

### 5. 可维护优先
- 可复用样式抽到 `css/style.css`
- 页面仅保留必要的局部结构和少量页面特有类

## 全站设计骨架

### 视觉气质
- 关键词：`编辑感`、`克制`、`轻科技`、`内容品牌`
- 主策略：`Content First`
- 风格组合：
  - 骨架：`Editorial / Magazine`
  - 交互：`Minimal & Direct`
  - 点缀：`Controlled Glassmorphism`

### 色彩策略
- 公共基础：浅色近纸面、深色近夜间阅读工作台
- 公共强调色：蓝色系，用于链接、按钮、选中态、功能状态
- 页面允许有频道色，但只作为点缀，不改变公共设计语言

### 组件层级
- `导航层`：最强玻璃感
- `工具层`：中高玻璃感
- `摘要层`：中玻璃感
- `内容层`：轻玻璃或纯净面板
- `正文层`：最克制、最清晰

### 动效策略
- 微交互时长控制在 `150-300ms`
- 减少持续性装饰动画
- 保留必要的进入动效、hover、focus 反馈
- 为 `prefers-reduced-motion` 提供降级

## 逐页方案

### 首页 `index.html`
#### 页面定位
- 作者门户首页

#### 目标
- 从“视觉开场页”升级为“全站入口页”
- 保留个人气质，但降低背景特效对主要内容的竞争

#### 改造要点
- 保留头像、主标题和一句身份定位
- 弱化过强 blob 与视觉噪声，保留更克制的氛围背景
- 在首屏下增加四联入口卡：
  - 最新文章
  - 生活瞬间
  - 收藏推荐
  - ACG 入口
- 标签区改为轻玻璃 pill 系统
- 导航改为悬浮玻璃导航，不贴顶边

#### 玻璃元素
- 强玻璃：导航、移动菜单
- 中玻璃：四联入口卡、作者状态摘要
- 弱玻璃：标签与次级提示

### 博客页 `blog.html`
#### 页面定位
- 编辑式内容入口页

#### 目标
- 从“文章列表”升级为“内容策展页”

#### 改造要点
- 保留标题与统计区，但提升区块间节奏
- 搜索区与视图切换器统一成玻璃工具条
- 在文章列表前加入摘要区：
  - 精选专题
  - 热门标签
  - 最近更新
- 文章卡突出：
  - 标题
  - 摘要
  - 元信息
  - 标签
- 降低页面纯装饰动画密度，改为更稳定的内容节奏

#### 玻璃元素
- 强玻璃：搜索工具条、视图切换区
- 中玻璃：摘要区卡片
- 弱玻璃：文章卡

### 文章详情页 `post.html`
#### 页面定位
- 深度阅读页

#### 目标
- 成为站内最稳定、最耐读的一页

#### 改造要点
- 保留头图 hero
- 正文区改造为更稳定的阅读面板
- 右侧目录与相关推荐统一为信息侧栏
- 强化标题、导语、引用、代码块、分节层级
- 控制非正文元素的视觉存在感

#### 玻璃元素
- 强玻璃：移动目录抽屉、顶部工具区
- 中玻璃：侧栏目录卡、相关推荐卡
- 正文主体：不做强玻璃，仅保留清晰阅读背景

### 瞬间页 `moments.html`
#### 页面定位
- 画册式生活流

#### 目标
- 从“功能集合页”升级为“生活画册页”

#### 改造要点
- 首屏压缩为更短、更明确的频道头部
- 图片流前置，成为页面主角
- 统计卡与热力图下沉到第二层内容
- 卡片布局强调：
  - 照片
  - 文本
  - 时间/地点
  - 标签
- 灯箱详情区更像作品详情层，而非纯预览层

#### 玻璃元素
- 强玻璃：筛选工具条、灯箱控制区
- 中玻璃：统计卡与热力图外框
- 弱玻璃：图片卡边框与层次

### 收藏页 `bookmarks.html`
#### 页面定位
- 策展型资源目录

#### 目标
- 从“资源工具页”升级为“知识策展页”

#### 改造要点
- 保留标题与搜索
- 增加编辑推荐摘要区或精选分类入口
- 强化分类导航层级，让分类区和资源区关系更清晰
- 资源卡突出“为什么值得收藏”，而不只是站点链接
- 提升空状态与无结果状态的内容感

#### 玻璃元素
- 强玻璃：搜索区、分类导航
- 中玻璃：精选入口卡
- 弱玻璃：资源卡

### RSS 页 `rss.html`
#### 页面定位
- 阅读工作台

#### 目标
- 从“功能页”升级为“内容阅读工作台”

#### 改造要点
- 保持三栏结构
- 明确三栏角色：
  - 左栏：订阅源目录
  - 中栏：文章收件箱
  - 右栏：沉浸阅读器
- 默认空状态更内容化
- 减少阅读区背景装饰干扰
- 设置弹窗与工具区样式统一

#### 玻璃元素
- 强玻璃：侧栏、顶栏、设置弹窗
- 中玻璃：默认空状态卡片
- 正文阅读区：保持低干扰

## 统一组件改造范围

### 导航栏
- 桌面端改为悬浮玻璃导航
- 移动端菜单统一玻璃抽屉语言
- 当前页高亮与 focus 状态统一

### Hero 区
- 统一标题层级、留白、次级说明和摘要卡节奏
- 保留页面差异，不统一成单一模板

### 工具条
- 搜索框、筛选器、视图切换器统一成玻璃工具条体系

### 卡片
- 统一圆角、边框、阴影、hover 反馈
- 内容卡轻、摘要卡中、浮层卡强

### 页脚与状态区
- 统一页脚结构与风格语言
- 缩小页面之间的重复实现差异

### 弹层与浮层
- 统一目录抽屉、移动菜单、RSS 设置等浮层风格

## 技术实现边界

### 优先修改
- `css/style.css`
- 六个页面的结构与类名组织
- 必要时少量调整 `js/main.js`

### 尽量不改
- 业务逻辑
- 数据加载流程
- 后端接口
- 后台管理系统

### 实现策略
- 公共样式优先
- 局部结构小步调整
- 不为视觉效果引入额外大型依赖

## 可访问性与体验要求
- focus 状态必须清晰可见
- hover 不应导致布局跳动
- 图片需要保留明确尺寸或固定比例容器，避免内容跳动
- 玻璃层在浅色模式下不可过透
- 文案与状态反馈保持一致，减少英文占位和杂乱 Loading 文案
- 支持 `prefers-reduced-motion`

## 验收标准

### 视觉验收
- 六个页面的导航、工具条、卡片、页脚属于同一设计系统
- 六个页面各自具备明确频道气质，不像同模板换文案

### 响应式验收
- 检查 `375 / 768 / 1024 / 1440`
- 不出现横向滚动
- 固定导航不遮挡核心内容

### 可用性验收
- 键盘可见 focus
- 搜索区、空状态、加载态具备明确反馈
- hover 与 active 反馈清晰但克制

### 技术验收
- 尽量不引入新的业务回归
- 修改后检查诊断问题
- 能跑的前端测试做针对性验证

## 实施顺序
1. 统一公共视觉层与导航体系
2. 改造 `index / blog / post`
3. 改造 `moments / bookmarks / rss`
4. 回头统一细节、状态、响应式与可访问性

## 风险与控制
- 风险：玻璃元素过多影响阅读
  - 控制：正文区不做重玻璃，仅工具区与摘要区强化
- 风险：样式统一后页面失去个性
  - 控制：通过模块编排与频道色差异化，而不是改基础骨架
- 风险：结构改动影响现有 JS
  - 控制：优先保留关键 ID 和现有数据容器
- 风险：移动端工具条过密
  - 控制：收敛为统一玻璃工具条，必要时折叠次要操作

## 结论
本次 UI 美化以“编辑感骨架 + 受控液态玻璃”为核心路线，目标是让核心公开页在保持内容优先与可读性的前提下，获得更统一、更高级、更有品牌感的体验。实现上优先走公共样式收敛与小步结构调整，不扩大到后台与业务逻辑层。
