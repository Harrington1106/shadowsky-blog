# Liquid Glass Homepage Redesign — Design Spec

- **Date**: 2026-06-24
- **Scope**: 主页（`index.html`）完全重构 + 全站导航栏样式统一
- **Status**: Approved (pending implementation)
- **References**: paveldogreat/WebGL-Fluid-Simulation, lusion.co, cloud-oc/liquid-portfolio, maorx.cn, lantxx.com.cn, uiverse.io/codebykay101/ugly-lion-23

---

## 1. 目标

把主页从"静态毛玻璃 + 模糊光晕"升级为**真正的液态玻璃**：

- 背景是**流动的霓虹液体**（WebGL Navier-Stokes 流体模拟），鼠标会推开它
- 内容极简：**一整面单屏（100dvh），零滚动，点开即全见**
- 中央三件套：头像 → 打字机循环诗句 → 介绍性 Tag 胶囊
- 顶部玻璃导航栏，底部底栏（含浏览量）
- 其他所有内容（笔记/片刻/收藏…）通过导航栏图标跳转到子页面，主页不堆
- **双主题**（暗色主推 / 亮色）完美适配
- **全设备适配**（移动/平板/桌面/超宽屏），UI 不重叠、不溢出

## 3. 流体背景（双主题适配）

基于 [PavelDoGreat Navier-Stokes 算法](https://github.com/PavelDoGreat/WebGL-Fluid-Simulation)精简改造（MIT 许可）。

### 3.1 流体行为

- **真实流体**：Navier-Stokes 速度场 advect 染料，鼠标拖动真的推开液体、留下霓虹拖尾
- **自主流动**：即使不碰鼠标，每 2-4 秒自动在随机位置喷一个色斑——背景持续流动
- **画布固定全屏**：`position: fixed; inset: 0; z-index: -1`，内容玻璃层浮在上面

### 3.2 双主题色板（关键适配点）

| | 暗色（默认/主推） | 亮色 |
|---|---|---|
| 底色 | `#060B18 → #0B1120` 深空黑蓝 | `#F5F7FA → #E8ECF1` 银白 |
| 流体霓虹 | 蓝`#3b82f6` / 紫`#8b5cf6` / 品红`#ec4899` / 青`#06b6d4`，高饱和发光 | 同色系**降饱和 + 提亮**，在白底上像水彩晕染 |
| 染料 alpha | 高（发光强） | 低（柔和不刺眼） |

### 3.3 主题切换响应

- 点主题按钮时，`fluid-bg.js` 监听 `<html>` class 变化
- **实时切换流体色板**，加 0.6s 颜色淡入过渡（不是突变）
- `navbar.js` 的 themeToggle 回调里 dispatch 一个 `themechange` 自定义事件通知流体引擎

### 3.4 性能分级

| 场景 | simResolution | dyeResolution | 预期表现 |
|------|--------------|---------------|---------|
| 桌面高端 | 0.5 | 1.0 | 60fps 满血 |
| 移动端 / 低端设备 | 0.25 | 0.5 | 30-60fps |
| 无 WebGL / `prefers-reduced-motion` | — | — | 降级到 CSS 动态渐变网格 |
| 标签页隐藏 (`visibilitychange`) | — | — | 暂停渲染循环 |

## 4. 视觉规格

### 4.1 玻璃质感（双主题）

| | 暗色 | 亮色 |
|---|---|---|
| 玻璃卡背景 | `rgba(17,24,39,.45)` + `blur(24px) saturate(160%)` | `rgba(255,255,255,.6)` + `blur(20px) saturate(140%)` |
| 边缘高光 | 顶部白 `rgba(255,255,255,.06)` + 彩色折射边 | 顶部白 `rgba(255,255,255,.7)` + 蓝紫折射边 |
| 中央聚焦区 | 柔和大玻璃圆角区，把三件套"托起" | 同结构，色值适配 |

### 4.2 字体

- 主体：Inter + Noto Sans SC（保留现有）
- **打字机诗句：`Noto Serif SC` 衬线**，增加诗意

### 4.3 动效

- **头像**：玻璃光环缓慢呼吸（4s loop）+ hover 时轻微 tilt
- **打字机**：打字 60ms/字 → 停留 2.5s → 删除 30ms/字 → 下一句，无限循环；光标 `_` 闪烁
- **Tag 胶囊**：入场逐个 stagger 弹入（每个延迟 80ms）；hover 上浮 + 发光
- **底栏浏览量**：绿点呼吸（复用现有 `pulse-dot`）+ 数字 count-up 从 0 滚到真实值

## 6. 全新导航栏样式（解决"丑" + "统一"）

### 6.1 问题诊断

现有 nav-link 每个都是独立的玻璃药丸（pill），7 个药丸挤一排 → 视觉碎片化、每个都"抢戏"，显得乱、丑。

### 6.2 新设计：一体式玻璃条 + 内嵌分区

```
现在(丑):  [◌首页] [笔记] [片刻] [收藏] [订阅] [ACG] [关于]  ← 7 个独立药丸
           每个都有自己的边框背景 → 碎片化

新设计:    ┌──────────────────────────────────────────┐
           │ ◉ ShadowQuake │ ◌首页  笔记  片刻  收藏 … │ 🌙 │
           └──────────────────────────────────────────┘
           ← 一个整体玻璃条，链接是条内凹槽，无独立边框
```

- **导航条本身**是悬浮玻璃（已有 `#navbar`），内部链接**去掉独立 pill 边框**，变成透明文字 + 图标
- **hover 态**：背景出现一个柔和 `rgba` 光块（无硬边框），文字微提亮
- **active 态**：底部一条 2px 霓虹渐变下划线 + 文字提亮，干净利落
- **图标缩小到 16px**，和文字 baseline 对齐，不再喧宾夺主
- **品牌区**和**链接区**用一条 1px 分隔线隔开，结构清晰

### 6.3 统一方式

样式全写在 `css/unified.css` 的 `#navbar .nav-link` 段。**改一处，15 个页面全部生效**（它们共用同一套 navbar 结构）：
- 主页 navbar：流体做 backdrop（透过玻璃看到流动液体）
- 其他页面 navbar：页面背景做 backdrop
- 结构样式完全一致，只是背景不同

## 7. 全设备适配

### 7.1 断点体系

| 设备 | 宽度 | 布局策略 |
|------|------|---------|
| 移动 | `< 640px` | 汉堡菜单；头像 72px；Tag 两行换行；底栏上下堆叠；流体 0.25 |
| 平板 | `640-1024px` | 汉堡菜单；头像 88px；Tag 单行；流体 0.35 |
| 桌面 | `> 1024px` | 完整 navbar；头像 112px；流体 0.5 |

### 7.2 单屏不溢出保障

- `min-height: 100dvh; overflow: hidden`
- 中央三件套用 `clamp()` 流式缩放 + flex 居中
- 头像 / 诗句字号 / Tag 全部 `clamp()`，任何屏幕都不溢出
- 窄屏下打字机文字自动换行，不压迫头像（gap 最小值约束）

### 7.3 移动端特色

- navbar 折叠成汉堡菜单（复用现有 `navbar.js` 逻辑）
- 头像缩小到 72px
- Tag 胶囊换行排列（flex-wrap）
- 底栏：浏览量和版权上下两行堆叠
- 流体分辨率降到 0.25，保 30fps+

### 7.4 超宽屏

- 中央内容 `max-width: 720px` 约束，不会无限拉伸
- 流体画布 `100vw × 100dvh`，视觉铺满

### 7.5 横竖屏切换

- `orientationchange` + `resize` 时流体画布 resize
- 中央聚焦区重新居中

## 9. 成功标准

1. 主页在任何设备上**一屏装满、不滚动**，点开即全见
2. 背景流体真实流动，鼠标推动产生涟漪拖尾
3. 双主题切换流畅，流体颜色实时适配
4. 导航栏在 15 个页面样式统一，不再碎片化
5. 移动端 30fps+ 不卡顿，无 WebGL 自动降级
6. 无内容重叠、溢出、错位
