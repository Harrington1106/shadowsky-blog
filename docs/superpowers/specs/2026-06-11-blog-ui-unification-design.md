# 博客页克制一体化收口设计文档

## 概述
- 项目：`shadowsky-blog`
- 日期：`2026-06-11`
- 主题：博客页视觉重收口
- 范围：`blog.html`、`css/style.css`、`js/core-pages-ui.test.js`、必要时 `docs/CODE_WIKI.md`

## 设计前提
- 用户已经明确否定当前博客页最近几轮“修补式”结果，核心反馈是：顶部仍割裂、搜索区难看、底栏难看，整体仍不像同一页。
- 用户本轮要求不再拘泥于已有局部方案，而是根据页面整体观感，直接判断哪里不好看并重做收口。
- 用户已确认新的视觉方向为“克制一体化”，即减少装饰噪音，让导航、标题区、搜索区、卡片区、底栏读起来像同一套页面系统。
- 仓库中仍未发现 `docs/DEVELOPMENT_MANUAL.md` 与 `project-state.yml`，本设计以现有代码、既有 spec/plan 与 `docs/CODE_WIKI.md` 为准。

## 用户确认结论
- 接受“克制一体化”作为博客页新方向。
- 不要继续堆玻璃特效、彩色 glow、额外壳层或看起来像修补补丁的视觉部件。
- 可以根据实际页面观感，主动判断博客页哪里不好看并直接修。
- 博客页导航应统一成首页那种完整文字胶囊导航，而不是保留当前图标栏。
- 底栏问题的重点不是“底栏本体太显眼”，而是页面尾部空白过大；后续方向必须是继续压低底栏、贴近分页、缩短页面结束距离。
- 搜索区大外框、分页按钮配色、视图切换标签边框、卡片配图观感都被明确判定为当前丑点，需要一起收口。

## 当前问题归纳

### 1. 顶部不是一个完整页面
- 导航、hero 和正文开头目前仍然像三块拼接区域。
- 页面最上沿与导航下方的背景连续性不足，导致首屏读感断裂。
- 任何额外叠加在导航外层的假壳、透明框、伪装饰，都会进一步强化“补丁感”。

### 2. 搜索区像独立组件，不像页面工具条
- 搜索框、光晕、切换胶囊之间仍有明显的“组件拼接感”。
- 当前搜索区太强调视觉效果，而不是内容操作本身。
- 用户已经明确指出搜索区“很丑”，说明这里只做数值微调已经不够。

### 3. 卡片区层级杂，内容感不足
- 博客卡片虽然已有一轮样式增强，但仍保留偏“漂浮盒子”的观感。
- 卡片和工具区、页面背景之间没有形成稳定的节奏关系。
- 当前卡片更像被特效托起来，而不是自然地排布在内容页中。

### 4. 底栏像页面之外的贴片
- 底栏目前仍容易被感知为“最后又加了一条东西”。
- 对博客页这种内容列表页而言，底栏应该是收尾，而不是另一个视觉主角。
- 如果底栏继续使用独立条带、渐变带、强边界线，会继续加重割裂。

### 5. 分页与视图切换器配色不协调
- “网格 / 时间轴 / 目录 / 标签”这排切换按钮像另一套边框系统，与搜索框和整体页面语言不统一。
- 分页按钮当前的色温、边框和高亮方式不高级，容易显得廉价。
- 这两组控件都需要更接近首页胶囊导航的克制配色与间距逻辑。

### 6. 卡片尤其是配图质量不稳
- 卡片本体的边框与阴影已经影响观感，但更明显的问题来自封面图质量参差。
- 当前配图如果比例、裁切、色调或清晰度不好，会直接把整张卡片拉低质感。
- 必须在不改数据源的前提下，用更稳定的封面比例、遮罩和兜底策略削弱坏图影响。

## 目标
- 让博客页顶部读起来像一个连续页面，而不是导航、hero、正文三段拼接。
- 让搜索区退回到安静、克制、清晰的工具条角色。
- 让卡片更稳定、更像内容本体，而不是高存在感玻璃块。
- 让分页按钮和视图切换器更接近首页导航那套克制胶囊语言。
- 让卡片封面即使遇到质量一般的图片，也不会破坏整页气质。
- 让底栏退回到页面结尾层，压缩尾部空白，消除“独立贴片感”。
- 保持博客页功能结构不变，不改数据逻辑，不新增模块。

## 非目标
- 不重做博客功能，不改 `js/blog.js` 数据流与视图切换逻辑。
- 不新增新的推荐区、标签面板、引导模块或动态特效。
- 不重做公开页共享导航文案合同，但博客页可改用首页同款公开导航结构与表现。
- 不修改 `post.html` 或其他公开页结构。

## 方案比较

### 方案 A：继续局部微调
- 做法：继续调导航、搜索、底栏单点样式参数。
- 优点：改动小。
- 缺点：仍会停留在“补丁感”，很难真正解决整页不统一的问题。

### 方案 B：克制一体化重收口（推荐）
- 做法：不改页面功能骨架，但统一顶部背景、工具区语气、卡片层级与底栏角色。
- 优点：改动集中、目标明确，能直接回应用户指出的真实观感问题。
- 缺点：需要回退掉前一轮部分花哨处理，重新定义博客页局部规则。

### 方案 C：彻底重做博客首页视觉
- 做法：连 hero、工具区和卡片布局关系一起重构。
- 优点：变化最大。
- 缺点：超出当前需求，风险偏高，也会增加额外回归范围。

## 采用方案
- 采用方案 B：在不改功能和数据流的前提下，对博客页进行一次完整的视觉重收口。

## 设计方案

### 1. 顶部连续化
- 博客页顶部不再出现“导航一层、hero 一层、正文一层”的三段式观感。
- `nav` 只保留真正需要的导航胶囊本体，不再额外增加假壳或透明方框。
- `hero` 与正文开始区域统一回到同一套页面背景系统中，重点是“连续”，不是“更亮”。
- 标题区本身继续保留，但存在感应弱于首页，不抢正文列表节奏。

### 2. 搜索区去组件化
- 搜索区不再保留难看的大外框，输入框本体直接成为主视觉。
- 搜索框与视图切换器归入一个安静的工具条系统，但两者层级分离，不再堆成一块厚板。
- 去掉或极大压低彩色 glow、炫光边缘、多余高光层，让搜索区更像内容页控制栏。
- 搜索框强调输入可读性和轮廓清晰度，而不是“特效感”。
- 切换胶囊的 active 态只保留必要反馈，不再成为页面第二主视觉。

### 3. 导航首页化
- 博客页顶部导航直接统一为首页那种完整文字胶囊导航。
- 站点标题、主导航项、主题切换的间距、圆角、边框、悬停反馈与首页一致，不再保留当前博客页的图标导航栏。
- 导航首页化是整页统一感的第一优先级，因为它决定了博客页与首页是否被读作同一套公开页系统。

### 4. 控件克制化
- 视图切换器收成细胶囊，不再使用厚边框和突兀底色。
- 分页按钮统一到首页系中性色和轻量高亮，当前页只保留清晰但克制的 active 态。
- 所有辅助控件都要服务内容列表，不再抢视觉中心。

### 5. 卡片内容化
- 卡片边界更稳定，但阴影更克制。
- 卡片 hover 保留轻微反馈即可，不再强调明显漂浮。
- 标题、摘要、元信息之间的层级应该比边框、阴影更重要。
- 卡片与卡片之间的间距节奏要统一，让页面读感更像编辑型内容列表。
- 封面图应统一更稳定的显示比例与裁切方式，避免不同来源图片把卡片顶部切得参差不齐。
- 对质量一般的封面图增加轻遮罩或兜底背景策略，让文字区仍然显得稳。

### 6. 底栏收尾化
- 底栏不再使用强独立背景条、强边界线或额外玻璃带。
- 底栏应该融入页面背景，只承担“收尾信息”职责。
- 底栏高度继续压低，贴近分页，减少分页到底栏之间、底栏到底部视口之间的冗余空白。
- 保留版权与访问量，但降低视觉存在感，让它自然结束页面。

### 7. 作用域与实现约束
- 所有新规则都限定在 `data-page="blog"` 作用域内。
- 优先回收最近新增但观感不佳的博客页局部补丁，而不是继续叠加新效果。
- 页面 DOM 结构尽量少动，以 CSS 收口为主；但导航允许直接对齐首页同款结构，因为这是用户明确指定的结果。
- `js/blog.js` 可在不改数据来源的前提下，微调分页按钮、卡片封面兜底类名或结构，前提是只服务当前视觉合同。

## 验收标准
- 博客页顶部不再出现明显的背景断层或补丁式壳体。
- 博客页导航与首页导航在结构和观感上保持同一套完整文字胶囊语言。
- 搜索区不再显得花哨和独立，且不再保留难看的大外框。
- 视图切换器与分页按钮的边框、配色、active 态不再突兀。
- 卡片观感更稳、更安静，内容层级优先于特效层级；问题配图对卡片质感的破坏明显降低。
- 底栏不再像页面最后单独贴上的条带，且尾部空白显著缩小。
- 页面整体观感比当前版本更克制、更完整。

## 风险与控制
- 风险：回退部分增强样式后，博客页可能显得过于平
  - 控制：保留必要的边界、轻阴影和 active 反馈，不走纯平路线
- 风险：博客页导航直接首页化后，可能与当前博客页 hero 间距不协调
  - 控制：同步重算博客页顶部留白和导航下方节奏，不只替换导航本体
- 风险：只做 CSS 收口可能不够
  - 控制：允许在 `blog.html` 与 `js/blog.js` 做极小结构与类名调整，但不动数据流
- 风险：博客页新规则继续污染其他公开页
  - 控制：所有规则必须带 `data-page="blog"` 前缀
- 风险：本次仍不涉及 PHP、SQL、Admin Token
  - 控制：无新增后端安全风险

## QA 审计报告

| 隐患点 | 严重程度 | 修复建议 |
|---|---|---|
| 顶部连续化如果处理不当，可能让导航和正文关系变平淡 | 中 | 用统一背景连续性替代额外装饰，而不是简单去样式 |
| 搜索区过度去装饰后可能失去操作焦点 | 低 | 保留清晰输入轮廓和轻量 active 态 |
| 导航首页化如果只替换结构不重算节奏，可能出现顶部拥挤 | 中 | 导航替换与顶部留白、hero 节奏一起调整 |
| 分页和视图切换器若继续沿用旧配色逻辑，会破坏整页统一感 | 低 | 一并统一到首页系中性色与轻 active 态 |
| 问题封面图仍可能拉低部分卡片质感 | 低 | 在卡片封面层增加统一比例、裁切和轻遮罩兜底 |
| 回退底栏特效后可能显得“像没设计” | 低 | 通过间距、字重和背景连续性来做收尾，而不是再加条带 |
| 本次不涉及 PHP 接口与数据库 | 低 | 无新增 SQL 注入风险 |
| 本次不涉及 Admin 鉴权逻辑 | 低 | 无新增 Token 校验风险 |

QA 结论：低风险，可进入新的实现计划阶段。

## 结论
这次博客页调整不再追求“更像玻璃”或“更有特效”，而是回到页面整体性本身。真正需要解决的是：顶部是否连续、搜索区是否安静、卡片是否像内容、底栏是否自然收尾。只要这四个问题被一起收掉，博客页就会从“到处在修”变成“像一页完整的东西”。
