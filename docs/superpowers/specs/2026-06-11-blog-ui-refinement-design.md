# 博客页 UI 收口与液态玻璃精修设计文档

## 概述
- 项目：`shadowsky-blog`
- 日期：`2026-06-11`
- 主题：博客页首屏收口、卡片精修、导航与底栏缝合
- 范围：`blog.html`、`css/style.css`、`js/core-pages-ui.test.js`、必要时 `docs/CODE_WIKI.md`

## 设计前提
- 当前博客页已经接入公开页液态玻璃系统，但真实截图显示博客页首屏仍存在明显的布局和层级问题。
- 用户已在真实页面中明确指出三个直接问题：
  - 删除博客页中间的 `blog-discovery` 发现面板；
  - 博客卡片需要进一步美化；
  - `footer` 底栏与页面背景存在割裂，导航栏可参考首页的一体化效果。
- 仓库中未发现 `docs/DEVELOPMENT_MANUAL.md` 与 `project-state.yml`，本设计以当前代码、已落地的公开页导航规范和 `docs/CODE_WIKI.md` 为准。

## 用户确认结论
- 删除博客页 `data-ui-toolbar="blog-discovery"` 这整块“先从这些内容开始阅读 / 精选专题 / 热门标签 / 最近更新”区域。
- 博客页继续保留标题区、搜索框、视图切换、正文列表、分页和底栏。
- 导航不用完全重写，但博客页的顶部导航视觉要向首页那种“一体化胶囊感”靠拢。
- 这次采用最小重构方案：不修改博客数据逻辑，不动 `js/blog.js` 的数据来源和列表渲染流程。

## 当前问题归纳

### 1. 首屏节奏失衡
- 标题区高度偏大，标题区和工具区之间留白过多。
- 用户打开博客页后，需要先穿过一段较空的 hero 区域，才进入真正的搜索与筛选操作。
- 删除发现面板后，如果不同时收口 hero 与工具区，博客页上半区仍会显得松散。

### 2. 发现面板与页面职责冲突
- `blog-discovery` 的文案和卡片属于“内容策展提示层”，但博客页本质上已经有搜索、视图切换和文章列表。
- 这块面板不仅拉长首屏，还和下面的文章列表形成重复导览。
- 用户已经明确要求整块删除，因此不保留任何简化版替代物。

### 3. 博客卡片不够精致
- 当前博客页卡片已经接入玻璃类，但仍偏通用卡片风格，缺少博客页专属层级。
- 从截图观感看，卡片与工具区、导航之间缺少统一的“材质语言”，更多是普通玻璃盒子的堆叠。
- 需要让博客卡片拥有更稳定的边缘、阴影、图文比例和 hover 反馈，而不是继续依赖强烈的彩色光晕。

### 4. 底栏与背景割裂
- 当前 `public-footer` 继承公开页共用玻璃底栏，但在博客页上下文里仍然像单独贴上的一条。
- 页面主体是浅灰背景，底栏却形成额外的一层边界，导致页面尾部收束不自然。
- 用户已经明确指出 `footer` 本身“有点问题”，需要在博客页做专门处理，而不是继续复用首页或其他页的默认尾部表现。

### 5. 博客页导航与首页导航风格不一致
- 首页导航已经形成较强的一体化胶囊感，而博客页导航仍更接近普通公开页浮动条。
- 这会导致用户从首页进入博客页后，感受到导航风格突然变薄、变散。
- 需要对博客页导航做页面级增强，但不能影响其他公开页导航合同。

## 目标
- 删除博客页发现面板，减少首屏信息噪音。
- 压缩博客页首屏节奏，让标题区、工具区和文章列表过渡更紧凑。
- 让博客卡片更有质感、更符合液态玻璃语境下的内容卡表现。
- 让博客页底栏与背景过渡更自然，不再像单独拼接层。
- 让博客页导航更接近首页导航的整体感，但不破坏全站共享导航系统。

## 非目标
- 不修改 `js/blog.js` 的数据加载、分页或视图切换逻辑。
- 不新增新的博客功能模块，不加入新的摘要区、推荐区或筛选器。
- 不修改其他公开页的 hero、底栏或导航结构。
- 不重做博客详情页 `post.html`。
- 不引入新的前端依赖或动画库。

## 方案比较

### 方案 A：只删发现面板
- 做法：删除 `blog-discovery`，其余保持不变。
- 优点：改动最小。
- 缺点：虽然删掉冗余块，但首屏留白、底栏割裂和卡片普通感仍然存在。

### 方案 B：推荐方案
- 做法：删除 `blog-discovery`，同步压缩 hero 与工具区间距，精修博客卡片，给博客页专门处理导航壳和底栏过渡。
- 优点：能一次性解决用户指出的三个真实问题，且改动集中在博客页与共享样式层。
- 缺点：需要新增博客页专用选择器，避免影响其他公开页。

### 方案 C：博客页整体重做
- 做法：重构博客页 hero、工具区、文章列表与分页的整套视觉结构。
- 优点：视觉变化最大。
- 缺点：超出当前用户要求，风险和回归范围都明显增大。

## 采用方案
- 采用方案 B：以最小重构方式解决博客页真实截图中暴露的核心问题。

## 设计方案

### 1. 删除发现面板
- 直接移除：
  - `section-shell`
  - `data-ui-section="blog-highlights"`
  - `data-ui-toolbar="blog-discovery"`
  - `data-ui-panel="blog-summary-grid"`
- 不保留任何占位块或过渡文案。
- 删除后，文章列表区直接接在工具区下方。

### 2. 收口博客页首屏
- 缩短博客页 hero 高度，降低现在 `min-h-[40vh]` 带来的空旷感。
- 压缩 hero 底部与主内容顶部之间的节奏。
- 工具区保留，但改成“首屏核心操作区”，让搜索框和视图切换成为博客页上半屏唯一的功能焦点。

### 3. 精修工具区
- 保留搜索框与视图切换，不调整功能顺序。
- 搜索框去掉过重的彩色 glow，改为更克制的边缘高光和轻阴影。
- 视图切换胶囊和搜索框在视觉上归入同一个完整工具面板，而不是上下两块松散组件。
- 工具区整体继续使用 `utility-glass-bar` 体系，但允许博客页专属补充类覆盖其边框、阴影和间距。

### 4. 精修博客卡片
- 目标是让博客卡片更像“内容卡”，而不是通用玻璃盒子。
- 重点增强：
  - 更稳定的外轮廓和边界
  - 更克制的阴影深度
  - 卡片 hover 时的轻微浮起与高光变化
  - 标题、摘要、元信息的层级清晰度
- 严禁重新引入强彩色泛光，避免首屏再次显得浮躁。

### 5. 博客页导航向首页靠拢
- 不改导航入口和交互，只做博客页专属外观增强。
- 目标是让博客页 `#navbar` 拥有更接近首页的连续胶囊壳体感：
  - 更完整的导航底板
  - 更统一的左右留白
  - 更清晰的壳体边缘和高光
- 该增强只作用于博客页，不能覆盖其他公开页。

### 6. 博客页底栏与背景缝合
- 博客页底栏不再保持当前过于独立的条带感。
- 重点处理：
  - 底栏背景透明度和边界线
  - 底栏上沿过渡
  - 与页面主背景的连续性
- 底栏仍保留版权文字和访问统计，不删除内容。

## 验收标准
- 博客页中不再存在 `blog-discovery` 发现面板。
- 博客页首屏留白明显减少，标题区与工具区衔接更紧凑。
- 搜索框与视图切换形成统一工具面板，不再显得割裂。
- 博客卡片视觉质感提升，但整体风格保持克制。
- 博客页底栏与背景过渡自然，不再像单独拼贴的一条。
- 博客页导航更接近首页胶囊感，但不影响其他公开页导航。

## 风险与控制
- 风险：删除发现面板后，博客页上半区信息过少
  - 控制：同步收口 hero 和工具区节奏，避免出现新的空白断层
- 风险：博客页专属导航样式污染其他公开页
  - 控制：所有导航增强都挂在博客页上下文选择器下
- 风险：卡片精修影响 `js/blog.js` 动态注入结构
  - 控制：只基于现有博客卡片类名和容器类名增强，不改数据渲染逻辑
- 风险：底栏改动影响全站共用 footer
  - 控制：优先使用博客页专属覆盖，不直接回退全局 `public-footer`

## QA 审计报告

| 隐患点 | 严重程度 | 修复建议 |
|---|---|---|
| 删除发现面板可能导致博客页首屏结构失衡 | 中 | 必须同步压缩 hero 和工具区间距 |
| 博客页专属导航增强可能误伤其他公开页 | 中 | 使用博客页上下文限定选择器 |
| 卡片美化可能与动态渲染卡片结构不匹配 | 中 | 先基于现有卡片 DOM 做增强，不改 JS 输出结构 |
| 本次不涉及 PHP 接口与数据库查询 | 低 | 无新增 SQL 注入风险 |
| 本次不涉及 Admin Token 校验 | 低 | 无新增鉴权风险 |
| 仅调整博客页结构和 CSS 样式 | 低 | 性能风险可控，避免引入重型动画 |

QA 结论：方案为低风险，可进入实施计划阶段。

## 结论
本次博客页调整的重点不是新增内容，而是收回多余结构，让博客页重新回到“标题 + 工具区 + 文章列表”的清晰节奏。通过删除 `blog-discovery` 面板、压缩首屏、精修卡片，以及对博客页导航和底栏做专门缝合，博客页可以明显减少割裂感，并更贴近首页已经建立起来的液态玻璃语言。
