# 工作流总览

## 万能命令

```
bash deploy.sh "做了什么"
```

所有改动都用这一条部署。自动完成版本号、git add、commit、pull、push。

---

## 各类内容怎么改

| 内容 | 在哪改 | 改完做什么 |
|------|--------|-----------|
| **博客文章** | Obsidian 本地写 .md | `bash deploy.sh` |
| **删/改文章信息** | admin 面板 → 博客文章 → 删除/编辑 | 自动 git commit, 本地 `bash deploy.sh` 同步 |
| **书签收藏** | admin 面板 → 书签收藏 | 直接生效，数据在服务器 |
| **随手拍** | admin 面板 → 片刻动态 | 直接生效 |
| **RSS 订阅** | admin 面板 → RSS 订阅 | 直接生效 |
| **视频推荐** | admin 面板 → 视频推荐 | 直接生效 |
| **追番追漫** | admin 面板 → 追番追漫 | 直接生效 |
| **站点设置** | admin 面板 → 站点设置 | 直接生效 |
| **页面代码** | 本地改 HTML/CSS/JS | `bash deploy.sh` |

---

## 两张图看懂

```
┌─────────────────────────────────────────┐
│  代码类（HTML/CSS/JS/博客文章.md）        │
│  本地改 → bash deploy.sh → 服务器更新    │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  数据类（书签/随手拍/订阅/视频/番剧等）    │
│  admin 面板改 → 直接存服务器 → 即时生效   │
└─────────────────────────────────────────┘
```

---

## 注意

- **改了代码**：必须跑 `bash deploy.sh` 才会上线
- **改了数据**：admin 面板改完就生效，不用部署
- **两边都改了**：先 `bash deploy.sh`（自动 pull 同步），再 push
- **刷新看效果**：`Ctrl+Shift+R` 绕过 Cloudflare 缓存
