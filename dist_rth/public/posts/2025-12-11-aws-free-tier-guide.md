> **简介**：AWS Free Tier（免费套餐）是目前云服务商中最大方的试用计划，提供 **12 个月** 的 `t2.micro` 或 `t3.micro` 实例。这对于学习 Linux、搭建小型网站或测试环境是极佳的零成本资源。

## 第一阶段：准备工作与资格验证

AWS 的注册门槛主要在于**身份与支付验证**。

### 1. 必备材料
*   **邮箱**：推荐使用 Gmail 或 Outlook。
*   **手机号**：支持中国大陆 `+86` 手机号。
*   **支付方式**：必须提供 **Visa** 或 **Mastercard** 信用卡/借记卡（卡面有相应标志）。
    *   *注意*：AWS 会进行 1 美元的预授权扣款以验证卡片有效性（后续会退还）。单纯的银联卡不支持。

### 2. 无信用卡的替代方案
如果你没有双币信用卡，可尝试以下途径：
*   **办理借记卡**：中国银行、工商银行等大行均支持办理 Visa/Mastercard 借记卡（Debit Card），门槛远低于信用卡，且同样被 AWS 接受。
*   **教育优惠**：如果你拥有 `.edu` 邮箱，可申请 **AWS Educate** 账户，无需绑定银行卡即可使用有限的云资源。

## 第二阶段：创建免费 EC2 实例（关键避坑点）

注册成功后，进入 EC2 控制台创建实例。每一步都必须小心，否则会产生费用。

### 1. 区域选择 (Region)
并不是所有区域都免费，且网络质量差异巨大。
*   **推荐**：**亚太地区 (东京)** 或 **亚太地区 (新加坡)**（连接国内速度较快）。
*   **备选**：**美国西部 (俄勒冈)**（线路较稳定）。

### 2. 镜像与实例类型
*   **OS (系统)**：选择 **Debian** 或 **Ubuntu**。务必确认下方标有 **"Free tier eligible"**。
*   **Instance Type (配置)**：
    *   必须选择 **t2.micro** 或 **t3.micro**。
    *   *警告*：只有带有 **"Free tier eligible"** 标签的型号才是免费的。

### 3. 密钥对 (Key Pair)
*   创建并下载 `.pem` 私钥文件。
*   **妥善保管**：这是登录服务器的唯一凭证，AWS 不会保存私钥，一旦丢失无法找回。

### 4. 存储 (Storage)
*   免费套餐包含 **30GB** 的 EBS 存储。
*   默认通常只分配 8GB，建议手动修改为 **30GB** 以最大化利用权益。

## 第三阶段：严防“反撸”扣费策略

AWS 是后付费模式，一旦超额直接从信用卡扣款。请熟记以下**“三不”原则**：

### 1. 不开第二台
*   **限制**：每月免费时长为 **750 小时**。
*   **计算**：1 台机器 × 24 小时 × 31 天 = 744 小时（刚好够用）。
*   **后果**：如果你同时开了 2 台，半个月就会耗尽额度，剩下的时间按小时收费。

### 2. 不闲置公网 IP
*   **限制**：AWS 的弹性 IP (Elastic IP) 只有绑定在**正在运行**的实例上才免费。
*   **后果**：如果你关机了 (Stop) 或者解绑了 IP 但没释放 (Release)，闲置的 IP 会按小时扣费。
*   **建议**：新手直接使用随实例启动分配的动态 Public IP 即可，无需申请 Elastic IP。

### 3. 不超流量上限
*   **限制**：每月 **100GB** 出站流量（Data Transfer Out）。
*   **后果**：超出部分按 GB 计费。
*   **建议**：仅用于个人学习、博客或代码测试。严禁用于大流量下载或多人共享代理。

## 进阶保障：设置账单报警 (Budgets)

为了睡个安稳觉，强烈建议设置预算报警：

1.  在控制台搜索 **"AWS Budgets"**。
2.  创建一个 **Cost budget**（成本预算）。
3.  设置阈值为 **$0.01** 或 **$1.00**。
4.  输入你的邮箱地址。

一旦你的账户产生任何费用（哪怕只有 1 分钱），AWS 都会立即发邮件通知你，让你能及时关停服务止损。
